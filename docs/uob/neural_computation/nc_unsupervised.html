Unsupervised Learning

Labelled data is limited (time consuming, requires expertise, label in every possible situation).

Unsupervised learning: From observations (XXX) to latent variables (ZZZ, infer), without labeled data.

Available data: x1,...,xN∼pdata(x)x_1, ..., x_N \sim \mathcal{p}_{data}(x)x1​,...,xN​∼pdata​(x)

Goal: Learn useful features of the data / Learn the structure of the data.

Used for:

  • Represent data to a lower dimension through PCA (compress image or audio)
  • Group data by similarity through clustering (text analysis, social media users, produce labelled data)
  • Estimate PDF through Probability Density Estimation (generate fake data)
  • Create style or fake images through generation / synthesise (synthesize shoe styles from contours just by giving a few examples, image enhancement)

Auto-Encoders

Given data we want the model to infer something about the content/characteristics, i.e., the latent variable. We encode the data first, then decode it. We use bottleneck layer in the middle to compress the data we're learning (otherwise, if it's of the same dimension, the parameters would only be the identity matrices)

Standard / Basic AE refers to AE with bottleneck

Reconstruction error - in most cases just MSE(Mean-Squared Error) which includes the parameters of the encoder fϕf_{\phi}fϕ​ and the decoder gθg_{\theta}gθ​. We minimize it to 0 when finding optimal parameters.

Lrec=1N∑d=1D(x(d)−gθ(d)(fϕ(x)))2\mathcal{L}_{rec}=\frac{1}{N}\sum_{d=1}^D(x^{(d)}-g_{\theta}^{(d)}(f_{\phi}(x)))^2 Lrec​=N1​d=1∑D​(x(d)−gθ(d)​(fϕ​(x)))2

ϕ∗,θ∗=argmin⁡ϕ,θLrec\phi^*,\theta^*=\operatorname*{argmin}_{\phi, \theta}\mathcal{L}_{rec} ϕ∗,θ∗=ϕ,θargmin​Lrec​

Compression forces the encoder to learn the most prominent features

Trade-off: Wider-bottleneck structure gives better construction & less compression.

Applications:

  • Compress data - and encoded movie could be stored on a server, a decoder could be stored on a cloud or user's computer which would allow fast streaming with reduced resources
  • Cluster data - encoders in a way cluster data due to the same decoder - it has to cluster latent variables if it wants to recreate similar representations for similar input data
  • Learn a classifier - we keep the encoder that produces cluster representation (tho not necessarily useful for classifier) and attach a classifier that learns the decision boundaries
  • Data synthesis - we keep the decoder and sampling a random latent variable (if it is within realistic space). Due to curse of dimensionality, vanilla AEs are NOT appropriate for this because a randomly sampled latent variable in high dimension can be sampled from an unrepresentative space (i.e., where no real data would be encoded to)

NOT trained for generate new fake data. -> sampling problems, gaps in input space.

Generative Model

"A generative model describes how a dataset is generated, in terms of a probabilistic model. By sampling from this model, we are able to generate new data.” --Generative Deep Learning, David Foster

We train them to perform probability density estimation

For frequency-based approximation techniques the number of samples needed to fill the space grows exponentially with the number of dimensions.

In high dimensions, the input space would basically be empty. => So difficult to do PDE directly. => Indirectly by enforcing samples from model to be similar to real data. (VAEs / GANs)

Variational Auto-Encoders

VAE - uses an encoder that maps each input to a small multivariate Gaussian (with uncorrelated dimensions) which is part of a larger prior distribution and a decoder which is forced to decode from that prior

pϕ(z∣x)=N(μϕ(x),σϕ(x)2)p_{\phi}(\mathbf{z}|\mathbf{x})=\mathcal{N}(\bm \mu_{\phi}(\mathbf{x}), \bm \sigma_{\phi}(\mathbf{x})^2) pϕ​(z∣x)=N(μϕ​(x),σϕ​(x)2)

Note: because standard deviation must be positive, we assume the raw output of an encoder is log of standard deviation log⁡(σϕ(x))\log (\bm \sigma_{\phi}(\mathbf{x}))log(σϕ​(x)). To get actually get it, we apply the exponent. Not using ReLU due to bad gradient around 0 thus neurons die quickly.

Stochastic Decoder - decoder which at each SGD iteration for each input draws a different encoded sample that is determined by the probability clusters and tries to reconstruct any sample, even the unlikely ones.

Overall loss function for VAE consists of 2 parts. Reconstruction loss helps to learn to create new realistic data and the regularizer takes care of the prior. We want to minimize the loss on average over all training data.

LVAE=Lrec+λLreg\mathcal{L}_{\text{VAE}}=\mathcal{L}_{rec}+\lambda\mathcal{L}_{reg} LVAE​=Lrec​+λLreg​

ϕ∗,θ∗=argmin⁡ϕ,θ(Ex∼pdata[Lrec]+λEx∼pdata[Lreg])\phi^*,\theta^*=\operatorname*{argmin}_{\phi, \theta}(E_{x\sim p_{data}}[\mathcal{L}_{rec}] + \lambda E_{x\sim p_{data}}[\mathcal{L}_{reg}]) ϕ∗,θ∗=ϕ,θargmin​(Ex∼pdata​​[Lrec​]+λEx∼pdata​​[Lreg​])

With reconstruction loss, we try to reproduce xxx for any z~∼pϕ(z∣x)\tilde z \sim p_{\phi}(z|x)z~∼pϕ​(z∣x) as opposed to directly encoded point fϕ(x)f_{\phi}(x)fϕ​(x). If Gaussians overlap, the loss is high. It learns means that cluster samples and small STDs that reduce overlap.

Ex∼pdata[Lrec]=1D∑d=1D(x(d)−gθ(d)(z~))2E_{x\sim p_{data}}[\mathcal{L}_{rec}]=\frac{1}{D}\sum_{d=1}^D(x^{(d)}-g_{\theta}^{(d)}(\tilde z))^2 Ex∼pdata​​[Lrec​]=D1​d=1∑D​(x(d)−gθ(d)​(z~))2

With regularization we prevent distributions to turn to points (with 0 std) - we minimize KL Divergence between predicted posterior z~\tilde zz~ and Gaussian N(0,I)\mathcal{N}(0, I)N(0,I). DKLD_{KL}DKL​ between 2 Gaussians. Causes means to be around 0 and STDs to cover prior.

Ex∼pdata[Lreg]=DKL[pϕ(z∣x)∣∣N(0,I)]=12(∑d=1D(μϕ(d)(x))2⏟minimized at 0+(σϕ(d)(x))2−2log⁡σϕ(d)(x)−1⏟minimized at 1)E_{x\sim p_{data}}[\mathcal{L}_{reg}]= D_{KL}[p_{\phi}(z|x) || \mathcal{N}(0, I)]=\frac{1}{2}\left(\sum_{d=1}^D\underbrace{(\mu_{\phi}^{(d)}(x))^2}_{\text{minimized at 0}}+\underbrace{(\sigma_{\phi}^{(d)}(x))^2-2\log \sigma_{\phi}^{(d)}(x)-1}_{\text{minimized at 1}}\right) Ex∼pdata​​[Lreg​]=DKL​[pϕ​(z∣x)∣∣N(0,I)]=21​​d=1∑D​minimized at 0(μϕ(d)​(x))2​​+minimized at 1(σϕ(d)​(x))2−2logσϕ(d)​(x)−1​​​

Re-parameterization - since sampling is not differentiable, we give a constant to the network ϵ∼N(0,I)\epsilon \sim \mathcal{N}(0, I)ϵ∼N(0,I) with which it simulates sampling z~=μϕ(x)+σϕ⊙ϵ\tilde z=\mu_{\phi}(x)+\sigma_{\phi}\odot \epsilonz~=μϕ​(x)+σϕ​⊙ϵ.

Applications:

  • Generation - synthesizing new data. Note that in space of Z~\tilde ZZ~ there may still be gaps due to small λ\lambdaλ encouraging separation of dissimilar data and because SGD may end up in local optimum.
  • Interpolation - given some z~\tilde zz~, move from obe to another and see what values get produced. They're a lot smoother with VAEs thant with vanilla AEs. Transitions are smooth due to overlapping Gaussians.
  • Alering - altering specific data features. A new z=z1+α(zt,avg−z1)z=z_1+\alpha(z_{t,avg}-z_1)z=z1​+α(zt,avg​−z1​) is produced by interpolating the main feature by an average of features with target characteristic (requires knowing which data has target characteristic). E.g., "fake hair".
  • Compression and Reconstruction - given xxx, we prezict z~\tilde zz~ and give the mean μϕ(x)\mu_{\phi}(x)μϕ​(x) to the decoder for high quality reconsruction. It's worse than AE because it uses regularization.

Generative Adversarial Network

GAN - a network which consistes of a generator GθG_{\theta}Gθ​ which produces data in the same fashion as VAEs and a discriminator which tries to match the model's density function pθ(x)p_{\theta}(x)pθ​(x) with the real data density function pdata(x)p_{data}(x)pdata​(x) by learning a binary classifier.

When finding the optimal parameters, we take the expected value of the log of sigmoid as our loss. Because the log is monotonically increasing, for ϕ∗\phi^*ϕ∗ we just maximize the probability to 1 when the data is real and minimize it to 0 when the data is fake and for θ∗\theta^*θ∗ we maximize the probability when the data is fake:

ϕ∗=argmax⁡ϕEx∼pdata(x)[log⁡Dϕ(x)]+Ex∼pθ(x)[log⁡(1−Dϕ(x⏟Gθ(z)))]⏞JGAN(θ,ϕ)\phi^*=\operatorname*{argmax}_{\phi}\overbrace{E_{x\sim p_{data}(x)}[\log D_{\phi}(x)]+E_{x\sim p_{\theta}(x)}[\log(1-D_{\phi}(\underbrace{x}_{G_{\theta}(z)}))]}^{J_{GAN}(\theta,\phi)} ϕ∗=ϕargmax​Ex∼pdata​(x)​[logDϕ​(x)]+Ex∼pθ​(x)​[log(1−Dϕ​(Gθ​(z)x​​))]​JGAN​(θ,ϕ)​

θ∗=argmin⁡θEz∼p(z)[log⁡(1−Dϕ(Gθ(z)))]\theta^*=\operatorname*{argmin}_{\theta}E_{z\sim p(z)}[\log(1-D_{\phi}(G_{\theta}(z)))] θ∗=θargmin​Ez∼p(z)​[log(1−Dϕ​(Gθ​(z)))]

where x∼pθ(x)x\sim p_{\theta}(x)x∼pθ​(x) <=> "prior" z∼p(z)=N(0,I)z\sim p_(z) = \mathcal{N}(0,I)z∼p(​z)=N(0,I)

In practice

  • the expectation is calculated for either a batch of real or fake data.
  • we can convert maximization to minimization by putting - in front of logs.
  • Use −log(D(G(z)))-log(D(G(z)))−log(D(G(z))) for θ∗\theta^*θ∗ due to gradients issues.

Results in:

ϕ∗=argmin⁡ϕEx∼pdata(x)[−log⁡Dϕ(x)]⏟LD,r(x)+Ez∼p(z)[−log⁡(1−Dϕ(Gθ(z)))]⏟LD,f(z)\phi^*=\operatorname*{argmin}_{\phi}E_{x\sim p_{data}(x)}\underbrace{[-\log D_{\phi}(x)]}_{\mathcal{L}_{D,r}(x)}+E_{z\sim p(z)}\underbrace{[-\log(1-D_{\phi}(G_{\theta}(z)))]}_{\mathcal{L}_{D,f}(z)} ϕ∗=ϕargmin​Ex∼pdata​(x)​LD,r​(x)[−logDϕ​(x)]​​+Ez∼p(z)​LD,f​(z)[−log(1−Dϕ​(Gθ​(z)))]​​

θ∗=argmin⁡θEz∼p(z)[−log⁡(Dϕ(Gθ(z)))]⏟LG(z)\theta^*=\operatorname*{argmin}_{\theta}E_{z\sim p(z)}\underbrace{[-\log(D_{\phi}(G_{\theta}(z)))]}_{\mathcal{L}_{G}(z)} θ∗=θargmin​Ez∼p(z)​LG​(z)[−log(Dϕ​(Gθ​(z)))]​​

Two-player Min-Max game (from Game Theory) - parameter optimization method where discriminator tries to maximize the accuracy of the classifier and the generator tries to minimize it.

Algorithm:

θ,ϕ←\theta, \phi \leftarrowθ,ϕ← random initialisation

for EEE epochs, repeat the below steps:

  1. Train discriminator for KKK (usually 1) iteration of SGD
  • Sample real xi∼Xtrainx_i\sim X_{train}xi​∼Xtrain​ and fake(prior of z) zi∼N(0,I)z_i\sim \mathcal{N}(0, I)zi​∼N(0,I) data
  • calculate expected(average of batch) loss LD(ϕ,θ)\mathcal{L}_D(\phi,\theta)LD​(ϕ,θ)
  • ϕ←ϕ−α∂LD(ϕ,θ)∂ϕ\phi\leftarrow \phi-\alpha \frac{\partial \mathcal{L}_D(\phi, \theta)}{\partial \phi}ϕ←ϕ−α∂ϕ∂LD​(ϕ,θ)​
  1. Train generator for 1 iteration of SGD
  • Sample prior distribution of z, zi∼N(0,I)z_i\sim \mathcal{N}(0, I)zi​∼N(0,I)
  • calculate expected(averaged) loss LG(ϕ,θ)\mathcal{L}_G(\phi,\theta)LG​(ϕ,θ)
  • θ←θ−β∂LG(ϕ,θ)∂θ\theta \leftarrow \theta - \beta \frac{\partial \mathcal{L}_G(\phi, \theta)}{\partial \theta}θ←θ−β∂θ∂LG​(ϕ,θ)​

If we assume that we can train a perfect discriminator, optimal DDD is:

Dϕ∗(x)=pdata(x)pdata(x)+pθ(x)D_{\phi^*}(x) = \frac{p_{data}(x)}{p_{data}(x)+p_{\theta}(x)} Dϕ∗​(x)=pdata​(x)+pθ​(x)pdata​(x)​

the optimization can be proved to be equivalent to minimizing the distance between the real data and fake data distributions, i.e., Jensen-Shannon divergence:

JGAN(θ,ϕ∗)=2DJS[pdata(x)∣∣pθ(x)]−2log⁡2J_{GAN}(\theta, \phi^*)=2 D_{JS}[p_{data}(x)||p_{\theta}(x)]-2\log 2 JGAN​(θ,ϕ∗)=2DJS​[pdata​(x)∣∣pθ​(x)]−2log2

DJS[pdata(x)∣∣pθ(x)]=0⇔pdata(x)=pθ(x)D_{JS}[p_{data}(x)||p_{\theta}(x)]=0 \Leftrightarrow p_{data}(x) = p_{\theta}(x) DJS​[pdata​(x)∣∣pθ​(x)]=0⇔pdata​(x)=pθ​(x)

Thus perfect D∗=0.5D^*=0.5D∗=0.5

In practice

  • We can't train a perfect discriminator (only KKK (not infinite) updates and SGD can result in local minima).
  • We use different loss functions for the generator. Because with the original log loss the network may stop learning from bad fake samples because it is sure they are bad (log loss is 0 which means no gradient). For good real samples we would get big gradients and we would try to improve network even more. But what we want is to learn from bad rather than good samples. We substitute:

log⁡(1−Dϕ(Gθ(z)))→−log⁡Dϕ(Gθ(z))\log(1-D_{\phi}(G_{\theta}(z)))\to -\log D_{\phi}(G_{\theta}(z)) log(1−Dϕ​(Gθ​(z)))→−logDϕ​(Gθ​(z))
